    1. What is Semantic HTML? 
        - HTML that describes the outline of the page via keywords e.g. <section>,<p>,<img...>, etc.
    2. What is HTML used for? 
        - The language browsers use to structure webpages.
    3. What is an attribute and where do we put it? 
        - Attributes are added to certain tags to add context, urls, or accessibilty.
    4. What is the h1 tag used for? How many times should I use it on a page?
        - The largest heading tag. Used for titles on webpages.
    5. Name two tags that have required attributes
        - a (anchor tag) requires href.
        - img (image tag) requires <src> url to show an image. You can also add an <alt> text attribute that displays when the image cannot.
    6. What do we put in the head of our HTML document?
        -metadata: title, scripts, style sheets
    7. What is an id? 
        -CSS added to an element that should be unique. Implemented via "element #id{}"
    8. What elements can I add an id to? 
        -any of them but it must be unique and assigned to only one.
    9. How many times can I use the same id on a page? 
        -Once 
    10. What is a class?
        -Attribute for HTML elements that aid in styling
    11. What elements can I add a class to? 
        -Any of them
    12. How many times can I use the same class on a page? 
        -As much as you want to. Follow your heart.
    13. How do I get my link to open in a new tab?
        - Add target="_blank" to an <a... > element. Add <...rel="noopener noreferrer"> to prevent tabnabbing.
    14. What is the alt attribute in the image tag used for? 
        - Lets the user know what the image would be showing if the image didn't load correctly.
    15. How do I reference an id?
        - #id{}
    16. What is the difference between a section and a div
        -section is a semantic description of a section for the layout of your page. A div is a section in an html document that doesn't semantically describe the page but still seperates the page to aid in styling or behavior.
    17. What is CSS used for? 
        -styling a webpage. 
    18. How do we select an element? Example - every h2 on the page
        - element{} e.g. h2{}, body{}
    19. What is the difference between a class and an id? - Give me an example of when I might use each one
        - class can be used multiple times. id can only be used once. 
           -Example of class: .coloredtext{ color: blue}
           -Example of id: #title{ font-size: 45px} 
    20. How do we select classes in CSS?
        - .class{}
    21. How do we select a p element with a single class of “human””?
        -.human p{}
    22. What is a parent child selector? When would this be useful? 
        - When an element with a selector has a nested element with another selector.
        - It can be useful when you need to apply style to every element but only when under a certain parent.
    23. How do you select all links within a div with the class of sidebar?
        - .sidebar a{}
    24. What is a pseudo selector?
        - Used to show different states of the element. e.g. hover, focused, etc.
    25. What do we use the change the spacing between lines?
        - Line-height
    26. What do we use to change the spacing between letters?
        - Letter-spacing
    27. What do we use to to change everything to CAPITALS? lowercase? Capitalize?
        - Text-transform: uppercase
        - Text-transform: lowercase
        - Text-transform: capitalize
    28. How do I add a 1px border around my div that is dotted and black?
        -div{ border: 1px dotted black }
    29. How do I select everything on the page? 
        - *{} if you mean selecting all elements.
        - if you mean select all in window, ctrl/cmd+a :P
    30. How do I write a comment in CSS?
        - /* comment */
    31. How do I find out what file I am in, when I am using the command line? 
        - pwd
    32. Using the command line - how do I see a list of files/folders in my current folder?
        - ls
    33. How do I remove a file via the command line? Why do I have to be careful with this? 
        - rm <filename.filetype>
        - You could remove the wrong thing.
    34. Why should I use version control? 
        - Let's you stay in sync with changes that coworkers are doing.
    35. How often should I commit to github?
        - When you make something that doesn't break.
    36. What is the command we would use to push our repo up to github? 
        - git push
